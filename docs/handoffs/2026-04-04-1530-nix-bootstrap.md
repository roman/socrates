# Handoff: Nix Infrastructure Bootstrap

**Date**: 2026-04-04 15:30

## What Was Done

Bootstrapped the Nix development infrastructure for Socrates as a prerequisite
to Phase 0.1 (tk reliability spike).

### Changes

- **flake.nix** — nixDir-based flake with flake-parts, devenv (pinned v2.0.6),
  and nixDir v3 inputs
- **nix/packages/ticket/** — Nix package for wedow/ticket v0.3.2, the bash tk
  script. Wraps runtime deps (jq, ripgrep, coreutils, grep, sed, awk, git).
  Installed as `tk` in PATH.
- **nix/devenvs/default.nix** — devenv shell providing tk, jq, nixfmt hook,
  and the nixDir Claude Code skill
- **.gitignore** — excludes devenv artifacts (result, .devenv/, .pre-commit-config.yaml)
- **CLAUDE.md** — project instructions with session discipline, commit
  conventions, design principles, and the bootstrap ratchet mechanism

### Key Decisions

1. **wedow/ticket (bash) over h2oai/tk (Go)** — bash version has full WORKPLAN
   command compatibility out of the box (create, ready -a, --tags, --parent).
   Go version is missing assignee filter on ready and has no tags support.
   May revisit h2oai/tk later (github.com/h2oai/tk).
2. **Pinned to v0.3.2 tag** — latest stable release. Plugins (query, ls, edit
   extracted to separate scripts) only exist on master post-v0.3.2. At v0.3.2
   these are all built into the main script, which is simpler to package.
3. **devenv pinned to v2.0.6 tag** — ensures cache hits from devenv.cachix.org.
   Removed nixpkgs follows override on devenv input for same reason.

## What's Next

- Run Phase 0.1 spike: create 20-30 tickets with deps, test CRUD, concurrent
  access, output parseability
- Phase 0.2 follows immediately (tk dependency support validation)

## Learnings

- devenv flake inputs should be pinned to tags, not branches, to get cache
  hits. Overriding nixpkgs follows on devenv also causes cache misses.
- wedow/ticket at v0.3.2 has all commands built-in (no plugin system yet).
  Plugin extraction happened post-v0.3.2 on master.

## Gaps

- No `.claude/` directory tracked — devenv creates it via the nixDir skill
  but it's generated, not committed
- ticket package doesn't run upstream tests (behave/python) — `dontBuild = true`
  skips them. Acceptable for a single bash script but worth noting.

## soc-dw00 — Build the spec-status CLI

Implemented the `spec` read-only CLI as a pure-bash script at
`plugins/socrates/templates/spec`, packaged via
`pkgs.writeShellApplication` in `nix/packages/spec-cli.nix`. The
binary is added to the dev shell via `nix/modules/devenv/socrates.nix`.

### Subcommands

- `spec` — usage banner, exit 0
- `spec status` — lists non-archived specs with phase, delimit state,
  and per-status task counts (draft/approved/poured)
- `spec tasks` — tabular listing with SPEC, TASK, STATUS, PRI,
  CATEGORY, TICKET, REVIEW columns
- `spec tasks --status S` — filter by status
- `spec tasks --review` — show only tasks with non-empty `<review>`

### Implementation notes

- Pure bash, no external tools beyond coreutils. Frontmatter parsing,
  phase detection, and `<review>` block scanning all use bash builtins
  (`case`, `[[ =~ ]]`, parameter expansion).
- `writeShellApplication` runs shellcheck at build time; a shellcheck
  pre-commit hook was already enabled in `nix/devenvs/default.nix`.
- Tests at `plugins/socrates/templates/spec.test.sh` use a fixture
  tree via `mktemp -d`, exercising: fresh spec (Describe DRAFT, no
  tasks), mid-phase spec (Delimit APPROVED, Direction DRAFT),
  poured spec (with `ticket:` field), review content detection,
  empty review, archive exclusion, and status/review filters.

### Code-critic findings

Spawned code-critic (opus). Two majors:
- M1: cancelled status not counted in `cmd_status`. Left as-is:
  the ticket's expected output explicitly shows only
  draft/approved/poured. Cancelled tasks are rare and can be filtered
  with `--status cancelled` in `spec tasks`.
- M3: test `fail` variable mixed boolean/accumulator. Fixed — all
  failure paths now use `fail=$((fail + 1))`.
